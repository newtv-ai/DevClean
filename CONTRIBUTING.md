# Contributing to DevClean

感谢参与 DevClean。磁盘清理代码的错误会造成真实数据损失，因此安全边界和可复现实验证据优先于功能数量。

## 开发流程

1. 先创建 issue，说明用户场景、受影响路径、预期证据和失败关闭策略。
2. 从小分支提交最小改动，不混入无关格式化或生成物。
3. 新增清理范围时，同时提交分类测试、拒绝测试、竞态/崩溃测试和文档更新。
4. 运行：

   ```powershell
   uv sync --frozen --python 3.13
   uv run --frozen --python 3.13 ruff check .
   uv run --frozen --python 3.13 mypy src/devclean
   uv run --frozen --python 3.13 pytest
   ```

5. PR 描述必须列出：用户可见变化、写入面、恢复方式、验证证据、未解决风险。

## 不可破坏的约束

- 扫描、分类、重复分析和 AI 导入不得修改扫描对象。
- 新扫描默认零选择；AI 输出不得自动选择或授权执行。
- 执行器不得接受目录、glob、任意命令或模型提供的路径。
- 受保护路径、系统维护范围、重解析点、云占位符和身份不完整对象必须失败关闭。
- 任何永久删除必须经过最终精确清单、强确认、持久意图和执行前身份复验。
- 不得用 `os.remove`、`unlink`、`rmtree` 或 Shell 回退绕过句柄执行器。
- 不确定的崩溃状态不得自动重放。

## 规则与适配器

第三方规则默认只能分类/预览。包含进程、PowerShell、注册表、删除、截断或任意脚本动作的规则不会被接受为执行权限。厂商适配器必须使用权威 API；不完整的降级结果只能扩大“保留/阻断”集合，不能扩大“可清理”集合。

## 安全问题

不要公开披露尚未修复的数据损失、路径逃逸或权限提升问题。按 [SECURITY.md](SECURITY.md) 的方式私下报告。

提交贡献即表示你有权按本项目 GPL-3.0-or-later 许可证提供该贡献。
