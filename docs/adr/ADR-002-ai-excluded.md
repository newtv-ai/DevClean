# ADR-002（历史）：受限 AI 审查包，而非 AI 文件系统控制

- 状态：**Superseded by [ADR-004](ADR-004-controlled-cleanup-workflow.md)**（原状态：Accepted）
- 日期：2026-07-12
- 被取代日期：2026-07-16
- 决策所有者：DevClean maintainers

## 历史背景

本 ADR 首次允许用户把已扫描缓存的有限元数据手工交给 Codex/Claude，并用随机批次 ID、严格 JSON 与本地身份复核限制模型输出。它正确确立了以下原则：AI 不是文件系统权限主体，不能返回路径、glob、命令或新候选；路径/元数据仍有隐私风险；schema 不能证明模型语义安全；模型回复不能写入 `ai_feedback` 后在未来自动执行。

原 Accepted 版本同时包含两项现已撤销的工作流：

1. 模型返回 `DELETE` 后可以直接触发本机永久删除尝试；
2. `UNSURE` 可以自动进入用户回收路径。

这两项把“建议”“选择”和“执行授权”混在一起，不符合当前扫描后受控清理架构，因此本 ADR 不再是规范性产品说明。

## 当前替代决策

[ADR-004](ADR-004-controlled-cleanup-workflow.md) 完整取代本 ADR 的具体协议与执行语义：

- 模型词汇改为 `KEEP`、`RECOMMEND_RECYCLE`、`UNSURE`，不存在 AI `DELETE`；`RECOMMEND_RECYCLE` 只能表示私有隔离建议；
- AI 导入对象的执行权限固定为 `NONE`，导入不修改选择、不生成批准计划、不调用执行器；
- 用户显式点击“采纳 AI 建议”是新的本地选择事件，随后仍需查看最终范围并键入确认口令；
- 外部模型不能选择执行模式；导入建议只能帮助用户定位候选，用户仍须在本地另行选择可恢复隔离或不可恢复清除；
- Windows Shell 回收站不再是产品恢复边界；默认可恢复动作以同卷私有隔离为终态，立即释放空间为 0；
- 不可恢复清除只对本地 capability 已判定可执行的精确文件开放，必须使用更强确认，并固定先隔离、核验/持久化不可逆意图，再从隔离区按句柄处置；不存在从原路径直接永久删除；
- 所有动作在扫描完成后发生，且必须先写持久意图、再执行、再对账；未知状态不自动重放。

## 仍然有效的原则

以下边界从 ADR-002 延续到 ADR-004：

- AI 只接收用户明确导出的当前批次候选元数据，不读取文件内容，不自动联网，不保存 API Key；
- 外部响应必须是严格、闭合、有界 JSON，精确覆盖当前随机 ID 集合；
- 未知/重复/遗漏 ID、错误摘要、额外字段、路径、命令、通配符和非标准 JSON 一律拒绝；
- 模型不能扩大本地扫描范围、选择批准根或改变 capability；
- 手工导出属于用户主动的数据披露，不能宣传为“AI 已认证安全”；
- 长期方向仍是用厂商精确适配器降低路径启发式和模型判断比例。

本文件仅保留决策历史。实现、测试、审计和用户文档引用工作流时必须以 ADR-004 为准。
