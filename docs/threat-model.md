# Reclaimer Windows GUI 清理威胁模型

## 当前产品边界

Reclaimer 是 Windows 11 本地磁盘清理工具。用户使用单文件 GUI EXE 扫描选定目录；GUI 不创建全盘 SQLite 索引，只保留有限的内存审查项。

当前存在三条文件变更路径：

1. **确定性自动清理**：仅当前用户 `%TEMP%` 或 `%LOCALAPPDATA%\\CrashDumps` 内、超过七天的普通文件。扫描过程中仅在全部硬条件满足时永久删除。
2. **AI 审查清理**：仅已扫描、命中已知开发缓存路径的精确文件。GUI 生成最多 50 项的脱敏元数据包；用户将 Codex 或 Claude 的固定 JSON 回复粘贴回 GUI。`DELETE` 只能作用于该 `review_id` 内已有的精确项，并在本机再次复核后永久删除；`UNSURE` 转入用户决定。
3. **用户决定**：用户勾选“需要你决定”的普通文件，GUI 完整显示路径并在 Windows 回收站操作前重验扫描快照。此路径可恢复，但受 Windows 回收站容量与系统策略限制。

不存在：目录递归删除、注册表清理、系统组件/Windows Update 清理、提权/UAC、服务启动、运行时插件、BleachBit `--clean`、自动上传、自动启动本地模型或任意外部命令执行。

GUI 还提供类别/顶层目录空间概览、大于等于 1 MiB 的有限 SHA-256 精确重复检测，以及上限 1 MiB 的脱敏 JSONL 操作历史。重复检测只把除一个基准文件外的副本转入用户决定，不会自动删除。

旧 CLI、SQLite 盘点和只读适配器仍用于开发兼容性；它们不是用户 GUI 的执行控制面。

## 不可突破的文件安全规则

所有删除路径都拒绝：目录、reparse point、Cloud Files placeholder、非固定本地卷、硬链接文件、缺失 128-bit file ID 或时间戳的文件，以及 `.git`、`.codex`、`.claude`、编辑器历史、`.env*`、私钥、证书和密码库。

永久删除必须：

- 使用扫描时取得的文件 ID、卷序列号、大小、属性、创建/修改时间和单硬链接快照；
- 以 `OPEN_REPARSE_POINT` 打开批准根与目标；
- 从已打开的文件句柄重新读取元数据，而不是进行第二次不受约束的路径读取；
- 验证最终句柄路径仍在普通批准根之下；
- 仅通过 `SetFileInformationByHandle(FileDispositionInfo)` 删除已复核句柄；不得回退到 `os.remove` 或目录删除。

文件在扫描后变化、被占用、失去身份信息或任何 Win32 调用失败时，都不得删除。自动路径转入用户决定；AI 路径也转入用户决定。

## AI 信任边界

AI 不是文件系统权限主体，也不能提供路径、命令、通配符、目录或新的候选项。审查包只包括脱敏路径、大小和既有分类原因，不包括文件内容。回复必须严格匹配当前批次的 schema、`review_id` 和全部 `item_id`；额外字段、漏项、重复项、未知 ID 或非标准 JSON 全部拒绝。

这只能限制 AI 的**作用范围**，不能证明语义判断正确。AI 的 `DELETE` 不是“文件绝对安全”的证明；它只是在用户选择的工作流中，对一个已受本机身份约束的缓存候选项作判断。云端模型还可能从路径、项目名、软件名和大小推断工作习惯，因此复制审查包属于用户明确发起的数据披露；GUI 不自动发送数据、不保存 API Key，也不启动模型。

## 主要威胁与控制

| 威胁 | 当前控制 | 残余风险 |
|---|---|---|
| 重解析点、junction、云占位符越界 | 扫描不下降；删除再次拒绝并在句柄上复核 | Windows 文件系统不存在跨文件原子事务，句柄复核仅缩小竞态窗口 |
| 路径替换/TOCTOU | 扫描快照 + 句柄元数据 + 最终路径校验 | 恶意本地高权限对手不在本产品能力范围 |
| AI 输出扩大范围 | 固定批次、固定 ID、严格 JSON、无路径字段 | 用户可手工粘贴错误或伪造的回复；只会影响当前精确候选集 |
| AI 语义误判 | 仅对已知缓存候选；保护目录硬拒绝；失败转用户栏 | 模型不能证明文件可重建性，后续应以工具专用适配器取代路径猜测 |
| 永久删除不可恢复 | 自动范围极窄；AI/自动均作句柄复核 | 成功的永久删除没有 undo；后续需要轻量历史、策略与可选回收站 |
| 回收站错误或容量不足 | 每项扫描快照重验；Shell 要求回收站与警告语义 | 大文件可能无法回收，用户需要检查 Windows 提示 |
| 重复检测误导用户 | 只哈希普通、稳定、非保护、单硬链接的大文件；每组自动保留一个基准 | 检测有容量上限，只覆盖大文件；用户仍需判断哪份副本应保留 |
| 操作历史泄露路径 | 路径替换为 `<USER>` 或 `<EXTERNAL>\\文件名`；不保存内容，文件上限 1 MiB | 历史不是完整取证账本，也不能恢复永久删除 |
| 机器扫描造成资源耗尽 | 流式扫描、取消令牌、每栏最多 500 个显示项 | 超大目录的 AI 候选超过上限时需按更小根目录继续处理 |
| 文档或旧 CLI 被误认为 GUI 行为 | 本文件、README 和实施状态定义 GUI 边界 | 旧 inventory 文档必须明确标注为开发兼容层 |

## 非目标与后续审查

Windows Update、`Windows.old`、`WinSxS`、`C:\Windows\Installer`、注册表、服务、应用卸载、WSL2 VHDX、目录批量删除和厂商 GC 都不属于当前自动删除范围。加入任一类前，必须先有专用分类器、明确的可恢复/不可恢复策略、预览、失败回退、canary 测试和独立审查。

本产品不把“释放空间”宣传为系统性能优化，也不提供注册表优化或反取证覆写功能。
