# DevClean 受控清理威胁模型

## 范围与安全目标

DevClean 是以当前标准用户权限运行的 Windows 11 本地 GUI。其目标不是证明任意文件“绝对可删”，而是在用户完成扫描后，为一小批精确文件提供有界、可审计、失败关闭的私有隔离或确认清除能力。

安全目标按优先级排列：

1. 扫描、分类、重复文件分析、AI 导出和 AI 导入不得改变任何扫描目标。
2. 默认零选择；AI 建议、文件类别和低风险标签都不能单独形成执行授权。
3. 最终动作不得超出完成扫描中的精确文件、用户可见选择和本地 capability 上限。
4. 身份、范围、根目录或持久状态出现不确定性时不执行，也不自动重放。
5. 硬保护资产不能被本地选择、AI 建议或批准根错误降级。
6. 审计事实区分“离开原路径并可恢复”“从隔离区不可恢复清除”和“已证明释放物理空间”。

## 信任边界

| 组件/输入 | 信任级别 | 能力 |
|---|---|---|
| 扫描器、分类器、增量监视器 | 观察层 | 只读文件系统元数据；不能导入执行模块 |
| AI 请求与响应 JSON | 不可信外部数据 | 只表达当前批次内的惰性建议；没有路径、命令或执行权限 |
| GUI 用户选择与最终口令 | 必要授权之一 | 只能缩小本地已批准候选；不能绕过 capability 和身份复核 |
| 本地 capability/硬拒绝规则 | 可信代码边界 | 决定执行上限；模型和导入数据不能修改 |
| SQLite 清理意图日志 | 持久安全状态 | 在副作用前记录批次/动作；没有自动 replay API |
| Windows 精确文件执行器 | 窄写入边界 | 仅处理普通文件的句柄绑定隔离、恢复和处置 |
| 同卷私有隔离区 | 可恢复写入边界 | 批准根直属目录在创建时即使用私有 DACL；保留精确对象及其原文件 DACL，不释放空间，不交给 Windows Shell 回收站 |

恶意管理员、内核驱动、能够修改 DevClean 进程内存或替换已加载代码的本地攻击者不在本模型可防御范围内。普通并发进程造成的重命名、替换、锁定和属性变化属于范围内威胁，必须失败关闭。

## 不可突破的产品不变量

- 扫描 worker 不引用执行原语；扫描开始不构成删除授权。
- 分类队列与执行策略是正交字段。`AI_REVIEW`、`DETERMINISTIC_CANDIDATE` 或“低风险”均不等于已选择。
- AI 响应只允许 `KEEP`、`RECOMMEND_RECYCLE`、`UNSURE`；导入后 `execution_authority` 固定为 `NONE`。其中 `RECOMMEND_RECYCLE` 的产品语义是“建议安全隔离”，不得调用 Windows 回收站。
- 本地 capability 先决定候选是否可执行。可执行候选默认仅允许可恢复私有隔离；不可恢复清除必须由用户在最终页主动选择，AI 建议不能选择、勾选或升级该模式。
- 一次用户计划为 1–256 个文件且不超过 1 TiB，内部拆成每批 1–32 个文件/256 GiB 的独立持久意图；用户对完整计划只做一次精确清单确认。不可恢复口令绑定计划摘要、模式、文件数、逻辑字节数、随机码与 10 分钟有效期。
- 所有不可恢复路径都必须先精确隔离、验证隔离后的同一身份并持久化不可逆意图，再从隔离区按句柄处置；不存在从原路径直接永久删除的快捷路径。
- 执行目标必须来自原始 `ScanRecord.root`，不能把候选父目录临时提升为批准根。
- 目录、reparse point、Cloud Files placeholder、非本地固定卷、硬链接数不为 1、文件 ID/时间戳不完整的对象不执行。
- `.git`、`.codex`、`.claude`、VCS/编辑器状态、用户文档与媒体目录、云同步目录、凭据/密钥/证书/密码库、系统关键目录和 DevClean 自身隔离/状态目录受运行期硬拒绝。
- 不存在 `os.remove`、`os.unlink`、`shutil.rmtree` 或宽范围命令 fallback。

## 主要威胁与控制

| 威胁 | 当前控制 | 残余风险/诚实边界 |
|---|---|---|
| 扫描过程中误删 | 扫描/分类/AI 层无执行依赖；闭环测试和发布静态检查禁止扫描 worker 调用执行器 | 仍需每个发布候选重新运行回归门禁；历史事故证据不能替代新构建验证 |
| 路径替换与 TOCTOU | 扫描快照；128 位 file ID、卷序列、大小、时间戳、链接数；句柄读取最终路径；按句柄移动/处置 | Windows 不提供跨整个批次的文件系统事务；任一目标失败会停止后续动作 |
| 隔离目录预占/接管 | 随机批次目录是批准根直属子目录；批准根句柄禁止 DELETE 共享；目录以最终私有 DACL 原子创建；已存在即拒绝 | 同一用户或管理员级恶意代码不在可完全隔离的信任边界内；任何竞争导致身份/共享冲突时失败关闭 |
| 批准根计算错误 | 原始扫描根 + 狭窄已知根双重约束；句柄复核；硬编码资产/系统 deny-list | 本地 capability 代码仍属于高风险可信计算基；扩展批准根必须独立审计 |
| reparse、云占位符或硬链接越界 | 扫描不下穿边界，候选构造和执行前再次拒绝；只允许单链接普通文件 | 新文件系统或未知属性组合按不可执行处理 |
| AI 输出注入路径/命令 | 随机包/候选 ID、摘要绑定、严格闭合 JSON、全量 ID 覆盖、大小/深度/文本上限、路径/命令检测 | schema 不能证明模型语义正确，因此建议仍需用户显式采纳并只能走本地既有上限 |
| AI 导入即自动删除 | 导入只生成惰性建议，不修改选择；“采纳 AI 建议”、选择不可逆模式和最终确认是三个本地用户事件 | 用户仍可能作出错误选择；模型不能替用户选择确认清除，也不能改变本地可执行资格 |
| 硬保护被分类器漏判 | 分类保护 + 执行器更宽的独立 deny-list；执行器保护优先且不可降级 | 新型凭据扩展名或非常规资产位置可能未被名称规则识别，仍依赖用户审阅 |
| 崩溃后重复执行 | 全批次意图在首个副作用前以 `synchronous=FULL` 提交；状态转换持久化；对账只观察，没有 replay API | `EXECUTING`/缺失对象可能只能标为 `UNKNOWN`，需要人工处理 |
| Windows Shell 回收可能静默永久化 | 产品不再调用 Shell 回收站；可恢复模式以私有隔离为终态 | 私有隔离仍占磁盘空间，必须诚实展示；GUI 支持查看/恢复，但不把隔离伪装成已释放空间 |
| 确认清除不可恢复 | 用户主动切换独立模式；强口令含文件数/逻辑字节；执行固定先隔离、再持久化不可逆意图、再精确处置 | 成功处置没有 undo；“可重下”不等于私有或自建内容可恢复 |
| 增量通知丢失 | 监视器只作 invalidation hint；溢出、序列/令牌异常、根变化、句柄丢失、容量/轮次超限均回退全量 | 当前只保证同进程会话；应用重启后必须全量扫描 |
| 旧结果覆盖新扫描 | GUI 后台任务携带会话令牌；只发布完整、未取消的原子结果 | UI 当前在刷新时保守清空选择；未实现跨刷新精细保留全部人工标记 |
| 隐私泄露 | 不发送文件内容/API Key；默认脱敏模型元数据；用户主动导出/复制 | 路径别名、项目名、软件名、大小和时间仍可能透露使用习惯，用户应审阅导出包 |
| 误报空间释放 | 私有隔离释放量固定为 0；确认清除只累计日志中已到 `PURGED` 的逻辑字节 | 稀疏/压缩文件、仍打开句柄和文件系统记账使逻辑大小不等于真实物理释放量 |

## 持久意图与故障语义

清理日志使用独立 SQLite 数据库：

```text
journal_mode=DELETE
synchronous=FULL
BEGIN IMMEDIATE
```

完整批次的所有 `INTENT_RECORDED` 必须先提交，之后才能开始第一个文件变更。可恢复隔离状态为：

```text
INTENT_RECORDED -> EXECUTING -> QUARANTINED
```

确认清除状态为 `INTENT_RECORDED -> EXECUTING -> QUARANTINED -> PURGE_PENDING -> PURGED`。`PURGE_PENDING` 必须在隔离身份验证成功后持久化；处置只针对隔离中的同一对象。异常可能成为 `FAILED_UNCHANGED`、`QUARANTINED` 或 `UNKNOWN`；恢复路径另行记录 `RESTORE_INTENT -> RESTORING -> RESTORED`。启动后对账只依据精确快照观察状态，不会把未知意图重新执行。

## 非目标

当前版本不提供注册表清理、服务控制、应用卸载、提权 broker、Windows Update/WinSxS/`Windows.old` 清理、WSL/Docker VHDX 压缩、目录递归删除、厂商 GC 执行、BleachBit `--clean`、运行时规则下载或内置模型/API 调用。

扩展任何执行范围前，必须新增版本化 capability、精确预览/对象边界、失败语义、恢复声明、真实 Windows canary、静态发布检查和独立安全复核。设计决策以 [ADR-004](adr/ADR-004-controlled-cleanup-workflow.md) 为准。
